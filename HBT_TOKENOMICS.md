### 전환 과정
```

→ 서버에서 스마트 컨트랙트 mint() 호출
→ 사용자 지갑에 HBT 전송
→ 블록체인 탐색기(basescan.org)에서 tx 확인 가능
```

## 📉 반감기 (Halving) — 비트코인 방식

> *"일찍 시작할수록 유리하다"*  
> *"비트코인처럼 무한 반감, 영원히 0에 수렴하지만 절대 0이 아님"*

### 핵심 원리
- ❌ 시간 기반 → ✅ **누적 채굴 발행량 기반**
- 사용자 적으면 → 구간이 오래 유지 → **초기 참여자 혜택**
- 사용자 폭증하면 → 구간 빠르게 소진 → **인플레이션 자동 억제**

### 반감기 구조

채굴 보상 풀: **60,000,000 HBT** (전체의 60%)

| 구간 | 발행 가능 | 전환 비율 (100P당) |
|------|-----------|-------------------|
| 1 | 30,000,000 HBT | 100 HBT |
| 2 | 15,000,000 HBT | 50 HBT |
| 3 | 7,500,000 HBT | 25 HBT |
| 4 | 3,750,000 HBT | 12.5 HBT |
| 5 | 1,875,000 HBT | 6.25 HBT |
| 6 | 937,500 HBT | 3.125 HBT |
| 7 | 468,750 HBT | 1.5625 HBT |
| ... | (무한 반복, 각 ÷2) | ... |
| **최소** | — | **1 HBT** (바닥) |

> 수학적 총합: 30,000,000 × 2 = **60,000,000 HBT**로 수렴

### 구간별 소진 시뮬레이션 (500명 매일 풀 인증 기준)

| 구간 | 월 발행량 | 소진 기간 |

## 🔥 소각 (Burn)

### 1. 챌린지 실패
- 예치 HBT의 **50% 소각**, 나머지 50% 반환
- 예: 1,000 HBT 예치 → 실패 시 500 HBT 소각, 500 HBT 반환

### 2. 외부 토큰 교환 수수료
- HBT → 외부 토큰 교환 시 교환량의 **2% 소각**
- 예: 10,000 HBT 교환 → 200 HBT 소각

### 소각 효과
- 총 공급량 감소 → 희소성 증가
- **반감기 + 소각 = 이중 디플레이션**
- 소각분은 복구 불가 (burn address 전송)

## 🔗 스마트 컨트랙트 구조

### 1. HaBit.sol — ERC-20 토큰
- mint, burn, transfer — 관리자만 mint 가능 (Ownable)
- 최대 발행량 100,000,000 HBT 하드코딩
- 소수점 8자리 (decimals = 8)
- 무한 반감기 로직 내장

```python
def getConversionRate():
    minted = totalMintedFromMining
    rate = 100
    threshold = 30_000_000
    while minted >= threshold:
        minted -= threshold
        threshold /= 2
        rate /= 2
    return max(rate, 1)  # 최소 1 HBT
```

### 2. HaBitStaking.sol — 챌린지 예치
- stake, withdraw, slash
- 성공 시 전액 반환 + 보너스 / 실패 시 50% 소각

### 3. HaBitExchange.sol — 외부 토큰 교환 (향후)
- HBT → 파트너 토큰 교환 또는 Uniswap V3 풀 활용
- 2% 소각 수수료 내장
